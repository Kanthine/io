/** 使用指定的选项创建一个基于位图的图形上下文
 *
 * 调用该函数来配置位图的绘图环境，位图的格式是使用 ARGB_32 整数像素格式。
 * 如果 opaque=YES, alpha 通道将被忽略，位图将被视为完全不透明(kCGImageAlphaNoneSkipFirst | kCGBitmapByteOrder32Host)。
 * 否则，每个像素使用预乘ARGB格式 (kCGImageAlphaPremultipliedFirst | kCGBitmapByteOrder32Host)。
 *
 * 使用 UIKit 默认坐标系统，原点在左上角，正轴向下延伸到原点的右边。所提供的比例因子 scale 也适用于坐标系统和生成的图像。
 * 绘图环境立即被推到图形上下文堆栈上。
 * 
 * 虽然这个函数创建的是当前上下文，但是可以调用 UIGraphicsGetImageFromCurrentImageContext() 函数来根据上下文的当前内容检索图像对象。
 * 当完成了上下文的修改后，您必须调用UIGraphicsEndImageContext() 函数来清理位图绘制环境，并从上下文堆栈的顶部删除图形上下文。
 * 您不应该使用 UIGraphicsPopContext() 函数从堆栈中删除这种类型的上下文。
 *
 * 在大多数其他方面，由该函数创建的图形上下文的行为与任何其他图形上下文一样。
 * 您可以通过推送和弹出其他图形上下文来更改上下文。
 * 您还可以使用 UIGraphicsGetCurrentContext() 函数来获取位图上下文。
 *
 * 这个函数可以从应用程序的任何线程调用。
 *
 * @param size 新位图上下文的大小(以点为单位)。表示UIGraphicsGetImageFromCurrentImageContext()函数返回的图像的大小。
 *             要获得位图的大小(以像素为单位)，必须将宽度和高度值乘以scale参数中的值。
 * @param opaque 指示位图是否不透明的布尔标志。如果你知道位图是完全不透明的，指定YES来忽略alpha通道并优化位图的存储。
 *               指定NO意味着位图必须包含一个alpha通道来处理任何部分透明的像素。
 * @param scale 应用于位图的比例因子。如果您指定一个值为0.0，则比例因子被设置为设备主屏幕的比例因子。 
 */
void UIGraphicsBeginImageContextWithOptions(CGSize size, BOOL opaque, CGFloat scale);

/** 获取当前图形上下文，可以从任何线程调用
 * 当前图形上下文默认为 nil，在调用 [view drawRect:] 方法之前，view 将一个有效的上下文推到堆栈上，使其成为当前的。
 * 如果没有使用 UIView 来绘制，必须使用 UIGraphicsPushContext() 函数手动将一个有效的上下文推到堆栈上。
 */
CGContextRef __nullable UIGraphicsGetCurrentContext(void);

/** 从堆栈顶部移除当前基于位图的图形上下文
 * 通过调用该函数来清理由 UIGraphicsBeginImageContext() 创建的绘图环境， 并从堆栈顶部删除相应的基于位图的图形上下文。
 * 如果当前上下文不是使用 UIGraphicsBeginImageContext() 函数创建的，则该函数什么也不做。
 * 这个函数可以从应用程序的任何线程调用。
 */
void UIGraphicsEndImageContext(void);
