MTLRenderCommandEncoder

MTLRenderCommandEncoder 用于渲染通道的命令编码器！

使用渲染命令编码器执行渲染传递:

  • 1、通过调用MTLCommandBuffer对象的renderCommandEncoderWithDescriptor:方法来创建一个mtlrendercommandencoderwithdescriptor对象，你想在这个对象中编码渲染命令。不要使用标准的分配和初始化技术来创建MTLRenderCommandEncoder对象。呈现命令编码器表示进入特定附件的单个呈现传递。通常，您只需创建一个MTLRenderPassAttachmentDescriptor对象来描述附件，然后重复使用它作为呈现命令的目的地，根据需要进行尽可能多的呈现。
  • 2、调用setRenderPipelineState:方法来指定一个MTLRenderPipelineState，该MTLRenderPipelineState定义了图形渲染管道的状态，包括顶点和片段函数。通常不经常创建这个MTLRenderPipelineState——例如，在初始化应用程序时。当您创建一个MTLRenderPipelineState时，您可以选择在mtlrenderpipelinerreflection中获取反射数据，该数据揭示了顶点函数和片段函数及其参数的细节。仅在需要时获取反射数据。
  • 3、指定顶点和片段函数的输入和输出资源。在相应的参数表中设置每个资源的位置(索引)。
  • 4、指定附加的固定功能状态，如视口、剪刀矩形、深度测试和模板测试设置。
  • 5、画的基本图形。
  • 6、调用endEncoding方法来终止呈现命令编码器。

要呈现多个镶嵌补丁的实例，请调用“绘制镶嵌补丁”部分中的任何方法。这些方法只能在顶点函数是后镶嵌顶点函数的情况下被调用;相反，你不能调用Drawing几何原语和Tessellated Patches部分中的任何方法。

配置图形渲染

/** 设置渲染管道
 *  @discussion 渲染管道默认为 nil，在发出绘图命令之前，必须设置好渲染管道；
 *              设置的渲染管道必须与 renderPassDescriptor 指定的 attachment 兼容
 *  @discussion 允许在编码周期内多次设置渲染管道：不同的场景可能有不同的数据、不同的渲染指令、调用不同的着色器
 *              更改渲染管道不会更改之前编码的任何命令
 */
- (void)setRenderPipelineState:(id <MTLRenderPipelineState>)pipelineState;

/** 如何栅格化三角形图元
 * MTLTriangleFillModeFill   填充三角形
 * MTLTriangleFillModeLines  仅画边
 */
- (void)setTriangleFillMode:(MTLTriangleFillMode)fillMode;

/** 用于确定正面图元的顶点绘制顺序：
 * MTLWindingClockwise 默认顺时针绘制顶点
 * MTLWindingCounterClockwise 逆时针绘制
 */
- (void)setFrontFacingWinding:(MTLWinding)frontFacingWinding;

/** 剔除图元
 * MTLCullModeNone 默认不剔除任何图元
 * MTLCullModeFront 剔除正面图元
 * MTLCullModeBack 剔除背面图元
 * @note 如果使用三角形图元定向填充一个 3D 球面，可以通过 图元的剔除策略、顶点的绘制顺序 实现隐藏背面仅显示正面的效果
 */
- (void)setCullMode:(MTLCullMode)cullMode;

配置深度和模板测试

/** 设置深度测试和模板测试，默认为 nil，不开启
 *  @note 开启深度测试或者模板测试，MTLRenderPassDescriptor 必须包含对应的深度附件、或者模板附件；且配置需要兼容
 */
- (void)setDepthStencilState:(nullable id <MTLDepthStencilState>)depthStencilState;

/** 通过缩放因子、缩放偏差调整片段着色器的深度值，将偏差控制在最大值
 * @param depthBias 对所有片段施加恒定偏差
 * @param slopeScale 与图元的深度梯度相匹配的偏差
 * @param clamp 应用于片段的最大偏差值
 * @note depthBias 作用于渲染管道的裁剪阶段；
 *       depthBias 仅影响三角形图元，而对点、线无影响
 */
- (void)setDepthBias:(float)depthBias slopeScale:(float)slopeScale clamp:(float)clamp;

片段着色器的参数设置

/** 为片段着色器设置一个数据块
 * @param bytes 数据块；如果数据小于 4kb，则使用该方法；如果超过 4kb，则必须使用 MTLBuffer
 * @param length 数据块的内存大小
 * @param index 在片段着色器的索引
 * @discussion 该方法 首先使用指定的数据块创建一个新的MTLBuffer对象，
 *             然后调用 -setFragmentBuffer:offset:atIndex: 将 buffer 绑定到片段着色器
 * @discussion 该方法避免了创建缓冲区来存储数据的开销；而是交由 Metal 管理数据
 */
- (void)setFragmentBytes:(const void *)bytes length:(NSUInteger)length atIndex:(NSUInteger)index;

/** 向片段着色器插入一个数据缓冲区
 * @param buffer 数据缓冲区
 *        缓冲区位于设备地址空间中，偏移量 offset 必须与片段着色器的数据类型对齐 (小于或等于16字节)
 *        缓冲区的地址空间为常量，在 macOS 中偏移量 offset 必须对齐到256字节；
 *           在iOS中，偏移量必须与 tile 着色器所消耗的数据类型的最大值或4字节对齐；如果不需要考虑数据类型，16字节对齐在iOS中是安全的；
 * @param offset 偏移量，以字节为单位
 * @param index 在片段着色器的索引
 */
- (void)setFragmentBuffer:(nullable id <MTLBuffer>)buffer offset:(NSUInteger)offset atIndex:(NSUInteger)index;

/** 向片段着色器插入一组数据缓冲区
 * @param buffers 缓冲区数据
 * @param offsets 一一对应的偏移量数据
 * @param range 插入的这组缓冲区，在片段着色器的纹理列表中的范围
 */
- (void)setFragmentBuffers:(const id <MTLBuffer> __nullable [__nonnull])buffers offsets:(const NSUInteger [__nonnull])offsets withRange:(NSRange)range;

/** 如果缓冲区已经绑定到片段着色器，该方法可以更改指定索引的缓冲区数据的偏移量
 *      比如调用 -setFragmentBuffer:offset:atIndex:     方法绑定单个缓冲区
 *      比如调用 -setFragmentBuffers:offsets:withRange: 方法绑定多个缓冲区
 *      比如调用 -setFragmentBytes:length:atIndex:      方法绑定数据
 * 如果不需要重新绑定数据、或者缓冲区，仅仅是修改其偏移量
 *
 * @param offset 新指定的偏移量，以字节为单位
 * @param index 在片段着色器的索引
 * @note 缓冲区位于设备地址空间中，偏移量 offset 必须与片段着色器的数据类型对齐 (小于或等于16字节)
 *       缓冲区的地址空间为常量，在 macOS 中偏移量 offset 必须对齐到256字节；
 *          在iOS中，偏移量必须与 tile 着色器所消耗的数据类型的最大值或4字节对齐；如果不需要考虑数据类型，16字节对齐在iOS中是安全的；
 */
- (void)setFragmentBufferOffset:(NSUInteger)offset atIndex:(NSUInteger)index;

/** 向片段着色器插入一张纹理
 * @param texture 待插入的纹理
 * @param index 纹理索引，默认指定索引的纹理为 nil
 *   每个片段着色器可以接收多个纹理，因此需要根据索引指明传入的纹理
 */
- (void)setFragmentTexture:(nullable id <MTLTexture>)texture atIndex:(NSUInteger)index;

/** 向片段着色器插入一组纹理
 * @param textures 待插入的一组纹理
 * @param range 插入的这组纹理，在片段着色器的纹理列表中的范围
 */
- (void)setFragmentTextures:(const id <MTLTexture> __nullable [__nonnull])textures withRange:(NSRange)range;

/** 为指定索引的纹理设置指定采样器
 * @param sampler 采样器，默认为 nil
 */
- (void)setFragmentSamplerState:(nullable id <MTLSamplerState>)sampler atIndex:(NSUInteger)index;

/// 向片段着色器插入一组采样器
- (void)setFragmentSamplerStates:(const id <MTLSamplerState> __nullable [__nonnull])samplers withRange:(NSRange)range;

/** 为指定索引的纹理设置指定采样器，并指定 min 采样、max 采样的级别
 * @param sampler 采样器，默认为 nil
 * @param lodMinClamp 采样级别
