卷绕与背面剪裁

1、背面剪裁

所谓背面剪裁，是指渲染管道在对构成 3D 物体的三角形图元绘制时，仅当观察点位于三角形正面的情况下才绘制三角形，若观察点位于背面则不进行绘制。

  • 开启背面裁剪，在大部分情况下可以提高渲染效率，去除大量不必要的渲染工作！
  • 对于封闭 3D 物体的渲染，一般情况下应该打开背面剪裁，避免渲染管道对被遮挡面的绘制，提高渲染速度；
  • 但如果绘制的是 2D 物体，希望正面和背面都被看到就不应该打开背面剪裁！

  • 打开背面剪裁后，正面朝向观察点的前面、上面、右面会被渲染管道绘制；而反面朝向观察点的左面、后面、下面则不会被绘制。

如果不打开背面剪裁，渲染管道会对所有的面进行绘制。被遮挡面上的片元虽然被绘制了，但还是会被遮挡面上的片元所覆盖，最终并不会出现在屏幕上。这就导致很多绘制工作是无用功，宝贵的计算资源被浪费，同样情况下应用程序的帧速率肯能会下降很多。

@protocol MTLRenderCommandEncoder <MTLCommandEncoder>

/** 背面剪裁
 * MTLCullModeNone 默认不剪裁任何面
 * MTLCullModeFront 剪切正面图元
 * MTLCullModeBack  剪切背面图元
 * @note 如果使用三角形图元定向填充一个 3D 球面，可以通过 图元的剪切策略、顶点的绘制顺序 实现隐藏背面仅显示正面的效果
 */
- (void)setCullMode:(MTLCullMode)cullMode;

@end

2、卷绕顺序

了解背面剪裁的功效后，还有一个问题就是 如何确定观察点位于一个面的正面还是反面。

在没有特殊设置时，Metal 默认从观察点观察一个三角形图元时：

  • 若三角形中的三个点卷绕顺序是顺时针，则该面是正面；
  • 若三角形中的三个点卷绕顺序是逆时针，则该面是背面；

当然，开发人员也可以通过设置以顺时针卷绕为背面，逆时针旋绕为正面！

@protocol MTLRenderCommandEncoder <MTLCommandEncoder>

/** 用于确定正面图元的顶点绘制顺序：
 * MTLWindingClockwise          默认顺时针卷绕为正面
 * MTLWindingCounterClockwise   设置顺时针卷绕为背面
 */
- (void)setFrontFacingWinding:(MTLWinding)frontFacingWinding;

@end

3、三角形示例

介绍完背面剪裁与卷绕之后，通过一个简单的 Demo 来验证其效果！

三角形顶点数据如下：

    static const AAPLVertex triangleVertices[] =
    {
        /// 卷绕：顺时针
        { {    0,   -50 }, { 0, 0, 1, 1 } },
        { {  100,  -150 }, { 1, 0, 0, 1 } },
        { { -100,  -150 }, { 0, 1, 0, 1 } },

        /// 卷绕：逆时针
        { {    0,   50 }, { 0, 0, 1, 1 } },
        { {  100,  150 }, { 1, 0, 0, 1 } },
        { { -100,  150 }, { 0, 1, 0, 1 } },
    };

3.1、不做背面剪裁

{
    id<MTLCommandBuffer> commandBuffer = [_commandQueue commandBuffer];
    MTLRenderPassDescriptor *renderPassDescriptor = view.currentRenderPassDescriptor;
    if(renderPassDescriptor != nil) {
        id<MTLRenderCommandEncoder> renderEncoder =
        [commandBuffer renderCommandEncoderWithDescriptor:renderPassDescriptor];

        [renderEncoder setFrontFacingWinding:MTLWindingClockwise];
        [renderEncoder setCullMode:MTLCullModeNone];

        ...
    }
}

在关闭背面剪裁的情况下，无论采用哪种卷绕方式，三角形都会被绘制！

3.2、背面剪裁

{
    id<MTLCommandBuffer> commandBuffer = [_commandQueue commandBuffer];
    MTLRenderPassDescriptor *renderPassDescriptor = view.currentRenderPassDescriptor;
    if(renderPassDescriptor != nil) {
        id<MTLRenderCommandEncoder> renderEncoder =
        [commandBuffer renderCommandEncoderWithDescriptor:renderPassDescriptor];

        [renderEncoder setFrontFacingWinding:MTLWindingClockwise];
        [renderEncoder setCullMode:MTLCullModeBack];

        ...
    }
}

开启背面剪裁的后，只有当下卷绕方式为观察点正面的三角形被绘制！

卷绕顺序与剪裁策略配合使用，可以实现不同的需求效果！
