Swift 类

1、类

在定义结构体时，声明的结构成员会在默认构造函数完成初始化；但是在定义一个类时，没有默认构造函数，因此如果不实现一个构造函数来初始化属性值，那么编译不通过！

class People {
    let name : String
    var age : Int    
}
/// Error 'People' has no initializers

可以在声明属性时，赋予一个初值；也可以实现一个构造函数;只有这样，一个类才知道如何初始化自己。

class People {
    let name : String = "人类"
    var age : Int = 5
}

class People {
    let name : String
    var age : Int

    init(age : Int, name : String) {
        self.age = age
        self.name = name
    }
}

类是引用类型!

类的定义与结构体定义相似！但结构体是值类型，而类是引用类型

var p1 = People(age: 10, name: "张三")
var p2 = p1 /// 指针 p2 指向类实例 p1 的内存地址：此时有 p1、p2 两个指针指向该块内存

p2.age = 11      /// 验证：修改 p2 的成员，p1 的成员也跟着改变；因为修改的都是同一块内存
p1.age == p2.age /// true

1.1、引用类型与值类型

值类型：通过赋值操作，会新开辟一块内存，将原有的值全部拷贝到新的内存空间； 此时虽然新变量与原变量相等，但它们已经分属于两个内存空间； 引用类型：一个指针指向一块内存空间，通过赋值操作一个新指针仍指向这块内存空间；也就是说，有两个指针都指向这个内存！

1.1.1、引用类型的特点

  • 在值类型，如结构体中，当一个结构体实例被声明为常量时，不论其成员是变量、常量；都不能再次修改；
  • 在引用类型，如类中，将一个类实例声明为常量，此时仍然可以修改变量类型的属性：
let p1 = People(age: 10, name: "张三")
p1.age = 11 /// 仍可修改

let 关键字约束指针 p1 ：

  • p1指针只能指向这块内存空间，不能再指向其它内存空间；
  • 但是在这块内存空间中，p1.age 却是一个变量，还可以修改；

在值类型中，一个变量不能修改其自身：

  • 一个值类型，每一次变化，都会产生一个新的值;
  • 在引用类型中，指针仅仅是指向该块内存，即使变化，仍指向该内存；
enum Switch {
    case On
    case Off

    func change() {
        switch self {
        case .On : 
            self = .Off ///Error : Cannot assign to value: 'self' is immutable
        case .Off :
            self = .On
        }
    }
}

在值类型中，自己改变自己，就会产生副本，那么这个副本放在哪里？与原始值有何关系？计算机不能自动处理！

Swift 的关键字 mutating 可以告知编译器：我要自己修改自己，修改完成之后副本覆盖原始值！

1.2、相等性

由于类是引用类型，那么引用类型如何判断相等性呢？

let p1 = People(age: 10, name: "Tom")
let p2 = People(age: 10, name: "Tom")
p1 == p2 /// Error: Binary operator '==' cannot be applied to two 'People' operands

== 是值类型之间的比较，不能用于引用类型！

1.2.1、指针指向的内存地址相等性

当然，如果想要判断两个指针是否指向同一块内存，那么可以使用运算符 === ：

/// 两次初始化，计算机分配了两块内存
p1 === p2 /// false
p1 !== p2 /// true

1.2.2、定义类的相等性

对于一个类而言，可以重载运算符 == 定义两个实例的相等性！

1.3、类与结构体

什么时候使用类？什么时候使用结构体？

结构体 类
将结构体体看作一个值 把类看作一个物体
描述位置、坐标、温度等数据 描述人、动物等
结构体更轻量级 类可以被继承
内存开辟在栈中 内存开辟在堆中

2、属性

Swift 中的属性可以分为以下几种类型：

  • 存储型属性：用于存储一个常量或变量；
  • 计算型属性：不直接存储值，而是通过get、set方法来取值或赋值。同时还可以对其他的属性进行操作；
  • 类属性：定义在类上的属性，用 static 来修饰属性，需要用类名来调用该属性。
struct Point {
    var x = 0.0
    var y = 0.0
}

struct Size {
    var width = 0.0
    var height = 0.0
}

class Rectangle {
    /// 存储型属性：存储一个值
    var origin = Point()
    var size = Size()
}

2.1、计算型属性

计算型属性：

  • 依赖其它属性而存在的属性；
  • 不需要存储一个值，仅仅存储如何计算该属性的规则；
  • 计算型属性不是一个常量，是根据其它值计算出来的，因此声明为 var；
  • 计算型属性需要显示声明其数据类型；
  • 计算型属性不能存储一个值：即使修改一个计算型属性，也是通过一定的逻辑，将该值的变化反馈到其它值中；
class Rectangle {
    var origin = Point() /// 存储型属性：存储一个值
    var size = Size()

    var area : Double { /// 只读权限
        return size.width * size.height
    }

    var center : Point { /// 读写权限
        get { /// 读操作
            return Point(x: origin.x + size.width / 2.0, y: origin.y + size.height / 2.0)
        }
        set { /// 写操作
            /// 计算型属性不能存储一个值，仅仅是通过一定的逻辑，将该值的变化反馈到其它值中
            origin.x = newValue.x - size.width / 2.0
            origin.y = newValue.y - size.height / 2.0
        }
    }
}

2.2、类属性

类属性：定义在类上的属性，用 static 来修饰属性，需要用类名来调用该属性。

class Rectangle {
    /// 类属性
    static var name = "矩形"

    /// 类方法，也称静态方法
    static func getName() -> String {
        return "类·方法"
    }
}

Rectangle.name
Rectangle.getName()

2.3、属性观察器

属性观察器：监测属性的变化！

  • willSet : 即将开始赋值
  • didSet : 赋值已经完成，拿到的值是新值；多用于保护数据的合法性
  • 注意：在初始化类实例阶段， willSet 与 didSet 并不会执行！
enum Theme {
    case DayMode
    case NightMode
}

class UI {

    /// 并不希望 color 值为 nil，但是在 themeMode 赋值之前，又无法确定 color 值
    /// 所以需要 color 值暂时为 nil，当用户真正使用 color 值确保一定有值，
    var fontColor : UIColor!
    var backColor : UIColor!
    var themeMode : Theme = .DayMode {
        willSet {
            print("属性即将改变: 旧值\(themeMode) - 新值\(newValue)")
        }

        didSet {
            print("属性已经改变: 旧值\(oldValue) - 新值\(themeMode)")
            self.changeTheme(themeMode: themeMode)
        }
    }

    func changeTheme(themeMode : Theme) {
        switch themeMode {
            case .DayMode:
                fontColor = UIColor.black
                backColor = UIColor.white
            case.NightMode:
                fontColor = UIColor.white
                backColor = UIColor.black
        }
    }

    init(themeMode : Theme ) {
        self.themeMode = themeMode  /// 注意⚠️：此时 willSet 与 didSet 并没有执行
        self.changeTheme(themeMode: themeMode) /// 初始化阶段，需要手动去为 color 赋值
    }
}

let ca = UI(themeMode: .NightMode) 
ca.themeMode = .DayMode

2.4、懒加载

  • 计算型属性：没有将计算出来的值存储起来，每次使用都用重新计算！
  • 懒加载：某个属性在使用的时候仅计算一次，计算之后存储该结果，方便下次使用；

使用 lazy 声明变量为延迟属性:

  • 必须显示声明其值类型，否则编译错误（此处的声明，可以看做声明一个闭包，必须有值类型）;
  • lazy 不支持 let ，仅支持 var
class UI { 
    /// lazy 语法是一个闭包
    lazy var viewTag : Int = {
        return Int(arc4random() % 999)
    }()
}

懒加载适用于：初始化实例的时候不需要为变量赋值，在后续使用的时候再赋值！

3、访问控制

Swift 中的访问控制以文件为单位：

  • private : 限制在本文件内访问；
  • public : 可以被模块外访问；
  • internal: 限制在本模块访问；

4、单例模式

Swift 的单例模式实现：

  • 1、将初始化函数设置为 private ，不允许外部访问；
  • 2、此时只有 UserManager 可以访问自身的初始化函数；
  • 3、因此通过类 UserManager 调用自身的初始化函数，使用 static 修饰将实例变量存储在常量区，保证仅初始化一次；
public class UserManager {
    static let shareManager = UserManager() /// 实例化之后，被存储在常量区，保证了仅初始化一次
    private init(){ /// 设置为 private

    }
}

5、面向对象的特性

面向对象的特性：如封装、继承、多态!

继承：子类可以拥有父类的属性与方法！

多态：不同的类实例调用同一方法可以获得不同的结果。实现多态的条件：

  • 必须存在继承关系；
  • 子类重写父类的方法；
  • 父类声明的变量指向子类对象；

重写(Override)：是子类对父类的允许访问的方法的实现过程进行重新编写, 返回值和形参都不能改变。即外壳不变，核心重写！

  • 重写的好处在于子类可以根据需要，定义特定于自己的行为。 也就是说子类能够根据需要实现父类的方法。

重载(Overload)：是在一个类里面，方法名字相同，而参数不同。返回类型可以相同也可以不同。

  • 每个重载的方法（或者构造函数）都必须有一个独一无二的参数类型列表。
  • 被重载的方法必须改变参数列表(参数个数或类型不一样)；

关键字 final:

  • 修饰某个类，不希望该类再被继承下去，终止继承关系！
  • 修饰某个方法，不希望该方法在子类中被重写；
class Book {

}

class MathBook: Book {

}

final class GeometryBook : MathBook {

}

5.1、两段式构造

Swift 的构造函数可以分为两部分：

  • 1、构造部分：用来初始化这个类; 此时 self 不存在，不能通过 self. 的方式调用未构造属性、所有的方法； 不仅要初始化本类独有的属性，还要初始化父类的属性；
  • 2、逻辑完善：直到父类的相关属性也构造完毕，才可以进一步完善该类，通过 self. 调用函数；

Swift 对子类的构造函数，涉及父类属性的初始化有一套严格的语法规定：

  • super要求：关于父类的属性，必须通过父类的构造函数构造；
  • 顺序要求：必须先初始化子类的所有属性，然后再调用 super 构造父类的属性
class Book {
    let author : String
    init(author : String) {
        self.author = author
    }
}

class MathBook: Book {
    let name : String
    init(author: String, name : String = "数学书") {

        self.name = name
        /// self.author = author /// 'self' used in property access 'author' before 'super.init' call
        super.init(author: author)

        /// 在没有完成整个类的初始化之前，不能调用类中函数
    }
}

构造函数本质也是一个函数，所以函数的诸多特性在构造函数也可以使用

  • 比如默认参数
  • 比如在子类重写该函数

5.2、便利构造函数、指定构造函数

init() 函数与普通函数不同，在构造函数中要想调用 init() 函数，需要指明是 self.init 还是 super.init!

便利构造函数，一般不执行实例化，仅仅是执行一些逻辑，然后将构造任务转给指定构造函数！

  • 在便利构造函数中，不能调用 super.init，仅仅通过 self.init 调用自身的指定构造函数；
  • 在便利构造函数中，不能通过 self. 为属性赋值，否则就是指定构造函数了；
  • 便利构造函数，最终一定要调用一个指定构造函数；在指定构造函数中，调用 super.init 来构造属于父类的属性；

class MathBook: Book {
    let name : String

    /// 指定构造函数
    init(author: String, name : String ) { 
        self.name = name
        super.init(author: author)        
    }

    /// 便利构造函数
    convenience init(dict:[String:String]){
        guard let author = dict["author"],
              let name = dict["name"] else {
            self.init(author: "", name: "")
            return
        }
        self.init(author: author, name: name)
    }
}

5.3、构造函数的继承

不同于结构体拥有默认的构造函数，声明一个类一般没有默认构造函数！但对于子类，某些时候可以继承父类的构造函数；子类构造函数的继承原则：

  • 如果子类没有实现任何父类的指定构造函数，则自动继承父类的所有指定构造函数；
  • 如果子类实现了父类所有的指定构造函数，则自动继承父类的所有便利构造函数！
class Book {
    let author : String

    /// 指定构造函数
    init(author : String) {
        self.author = author
    }

    /// 便利构造函数
    convenience init(dict:[String:String]){
        guard let author = dict["author"] else {
            self.init(author: "")
            return
        }
        self.init(author: author)
    }
}

class MathBook: Book {
    let name : String

    /// 指定构造函数
    init(author: String, name : String) {        
        self.name = name
        super.init(author: author)
    }

    /// 子类实现父类的指定构造函数
    convenience override init(author: String) {
        self.init(author: author, name: "数学书")
    }
}

/// 自动继承父类的便利构造函数
MathBook(dict: ["name" : "name"])

5.4、构造函数的继承

某些场景，可以使用 required 明确指定子类继承父类的构造函数!

  • required 构造函数，在子类声明中不需标注 override，而是标注 required；
class Book {
    let author : String
    required init(author : String) {
        self.author = author
    }
}

class MathBook: Book {
    let name : String

    init(author: String, name : String) {
        self.name = name
        super.init(author: author)
    }

    /// 需要声明为 required 构造函数
    required init(author: String) {
        self.name = "数学书"
        super.init(author: author)
    }
}
