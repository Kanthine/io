Swift 闭包

闭包本质上也是一个函数，是一个匿名函数！

1、闭包的声明

  • 闭包本质上是一个函数，因此可以传递函数型变量的参数，也可以传递一个闭包
  • 闭包就是一个匿名函数，因此声明一个闭包无需再写函数名，直接使用 () 声明参数即可
  • 闭包只在内部使用，因此只有内部参数名，没有外部参数名的概念；
var names = ["Tom","John","Tim"]

/// 使用 {} 声明一个闭包，{} 体中的内容就是闭包的内容
names.sort(by: { (a:String, b:String) -> Bool in
    return a > b
})

2、闭包的声明语法简化

Swift 是一门类型安全的语言，在传递函数型变量的地方使用闭包：

  • 函数型变量已经规定了参数名称、返回值类型等；如果没有符合其参数类型、返回值类型，就会编译失败；
  • 因此闭包中的参数、返回值类型，都没有必要再去声明；在语法上，这些声明都是冗余的；
  • Swift 会自动的为闭包中没有声明的参数、返回值附上类型；
var names = ["Tom","John","Tim"]

/// a, b 一定是 String 型，返回值一定是 Bool 型
names.sort(by: { a, b in return a > b })

/// 此处一定有返回值，那么 return 关键字也可以省略
names.sort(by: { a, b in a > b })

闭包的参数声明类似于元组的使用:

  • 在元组中可以为各个分量命名，也可以省略命名直接使用索引的方式访问指定变量
  • 在闭包中，如果不愿意为参数命名，Swift 也有默认命名 $0 $1 $2 ...
/// 在闭包中，使用默认参数名
names.sort(by: { $0 > $1 })

由于 sort() 函数只需要一个比较规则，因此可以直接传递一个比较符进去

///  > 也是一个函数，直接使用比较符
names.sort(by: >)

3、结尾闭包Trailing Closure

结尾闭包Trailing Closure：当闭包是一个函数需要传入的参数中的最后一个参数时，可以将闭包体放到参数列表外部！

/// 将闭包 {} 直接写到 () 之外
names.sort(){a, b in
    return a > b
}

/// 由于 sort 只有一个参数，因此将闭包提出来之后，参数列表 () 没有存在的必要
names.sort{a, b in
    return a > b
}

map() 函数中的内容，也是一个闭包

  • map() 闭包将数组的每一个元素，做一次规则转换；具体什么规则由开发者自定义；
  • 比如将 Int 型元素转为 String 型元素；
  • Swift 无法确定 map() 闭包的返回值类型， 所以返回值类型不能省略，必须显式指定；
var nums = [1, 2, 3, 4, 5, 6]
let numsStr = nums.map{ (num) -> String in
    return String(num)
}
